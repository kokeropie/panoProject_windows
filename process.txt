STEP 5 — reportDate
----------------------------------------
Source column: dataFilter/reportDate.txt (flat If/then/else chain, not
                nested — 6 sequential conditions, each returning another
                column's value rather than a literal)
Keyed on:       productNew (Step 2), Voided, DateOfCreated, VoidDate,
                DATEOFSERVICESTART — must run AFTER Step 2 (needs
                productNew)
Logic:          non-SPECIALIST rows use DateOfCreated (or VoidDate if
                voided); SPECIALIST rows use DateOfCreated/VoidDate only
                if it's later than DATEOFSERVICESTART, otherwise they use
                DATEOFSERVICESTART itself
Assumption:     the rule file compares "Voided = 1" for the voided case,
                but this dataset encodes voided rows as Voided = -1 (a
                legacy true/false convention), not 1. Treated "Voided = 1"
                as "Voided <> 0" so the voided branch still fires
                correctly — confirmed against the 3 voided rows, which
                all correctly picked up VoidDate as reportDate.
Default:        none — the 6 conditions are exhaustive over
                (productNew = SPECIALIST or not) x (voided or not) x
                (date-vs-DATEOFSERVICESTART); 0 nulls in this dataset.

file already exists at the target path it's deleted first, then the new

merges/appends, and fails with a clear "close it in Excel" error instead
of a bare PermissionError if the target is locked open.

STEP 9 — verify_outputs() (new, applies to every run)
----------------------------------------
Runs right after Step 8, before any file is written. Sanity-checks the
derived output and returns a list of OK/WARN lines (never raises — these
are data sanity checks, not code invariants):
  - output column count == source columns + 10 derived columns
  - output row count == source row count
  - null count per derived column (businessUnit, subBusinessUnit,
    productNew, subProduct, Channels, subChannels, Market, subMarket,
    reportDate, mgmtRpt)
  - output_void.xlsx row count matches rows where the Voided column
    (config voided_column) looks voided
  - output_active.xlsx row count matches output.xlsx
  - output_all.xlsx row count == active + void
  - for each configured negation_column, that the void-row magnitude
    matches the corresponding active row
Written to <outdir>/verification_report.txt alongside the 4 workbooks;
also logged line-by-line (WARN lines via log.warning). Surfaced in the
Streamlit app under a "Verification report" expander (auto-expanded if
any check warns).

STEP 10 — sanitize_commas() (new, applies to every run)
----------------------------------------
Runs after Step 9, right before writing. Every text/object column in all
4 output DataFrames has ',' replaced with ' ' in every string cell
(numbers, dates, and NaN are left untouched). Applied identically by the
CLI (pipeline.py) and the Streamlit "Run Pipeline" page (app.py).


SCHEMA-ADAPTATION FEATURES (new, config-driven, no-op on original.xlsx)
----------------------------------------
Added to support inputs shaped differently from original.xlsx (see the
Detail.xlsx run below) without touching the original.xlsx code path.
All three are configured in rules/config.json and are no-ops when the
condition they check for doesn't apply — verified against original.xlsx
above (589/589/3/592 rows, 0 warnings, identical to the pre-change log).

  - drop_incomplete_rows(): drops rows where config.required_nonnull_column
    (InvoiceID) is null, before any derivation step runs. original.xlsx
    has 0 such rows (no-op there).
  - ensure_voided_column(): if config.voided_column (Voided) isn't in the
    input at all, derives it (1/0) from config.voided_fallback_source_column
    (VoidDate) being non-null. original.xlsx already has Voided (no-op).
  - column_fallbacks in config ({"PCID": "newPCID"}): build_column_resolver()
    only applies a fallback when the canonical column (PCID) is genuinely
    absent from the input, so original.xlsx (which has real PCID) is
    unaffected.
